CS Interests & Research

Parallel Computing

Parallel and high-performance computing are the future. With the vast amount of data and information that we have to process, modern machines are not up to the task on their own. This is where distributed networks and parallel programming comes into play. Working closely with the hardware (such as CUDA with NVIDIA), software engineers can achieve peak performance. I, for one, wish to delve further into this powerful science.

Embedded Software Engineering

Tons of devices run computers such as phones, tablets, and gaming consoles. However, computers are used in far more areas than just these; big machines such as cranes, cars, home appliances, and many more all use computers, and software engineers have to write their programs to accomodate them. Often, one must write code on extremely limited hardware, which may also exist in harsh temperatures/weathering. Optimization is the key here, and I want to learn more.

Compiler Design

Compilers have existed in Computer Science since the earliest days of programming. The machine does not understand english words, so, the compiler has to translate the programmer's high-level syntax to a numeric [binary] representation. The Java Virtual Machine is an example of a very complex compiler. C, C++, and others are all compiled languages. Furthermore, they're one of the lowest levels that source code reaches before being fully interpreted by the machine.
